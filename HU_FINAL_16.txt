HU-01 – Crear Convocatoria
Rol: Coordinador Académico
Quiero crear una convocatoria de monitoría especificando curso, semestre y requisitos
Para que los estudiantes puedan postularse

Criterios de Aceptación:
- Dado que el coordinador esté autenticado, cuando ingrese al módulo de convocatorias y seleccione “Crear convocatoria”, entonces el sistema debe permitir ingresar curso, semestre y requisitos obligatorios.
- Dado que se cree una convocatoria, cuando se guarde el registro, entonces la convocatoria debe quedar disponible en el listado público para los estudiantes.
- Dado que se cree una convocatoria, cuando falte algún campo obligatorio, entonces el sistema debe impedir el guardado y mostrar un mensaje de error.

Dependencias:
HU-02, HU-03

------------------------------------------------------------

HU-02 – Asignar fechas de apertura y cierre
Rol: Coordinador Académico
Quiero asignar una fecha de apertura y cierre a cada convocatoria
Para que el sistema gestione el periodo de postulación

Criterios de Aceptación:
- Dado que se edite una convocatoria, cuando se asignen fechas de apertura y cierre, entonces el sistema debe validar que la fecha de cierre sea posterior a la de apertura.
- Dado que llegue la fecha de apertura, cuando el sistema procese la convocatoria, entonces esta debe mostrarse como disponible para postulación.
- Dado que llegue la fecha de cierre, cuando el sistema procese la convocatoria, entonces esta debe pasar automáticamente a estado cerrado.

Dependencias:
HU-01, HU-04

------------------------------------------------------------

HU-03 – Mostrar convocatorias activas
Rol: Sistema de Gestión Académica
Quiero mostrar las convocatorias activas en un módulo de consulta
Para que los estudiantes puedan revisarlas antes de postularse

Criterios de Aceptación:
- Dado que existan convocatorias activas, cuando un estudiante consulte el módulo, entonces el sistema debe mostrar todas las vigentes.
- Dado que una convocatoria llegue a su fecha de cierre, cuando se actualice el listado, entonces esta debe desaparecer del módulo de activas.
- Dado que no existan convocatorias activas, cuando un estudiante consulte, entonces el sistema debe mostrar un mensaje indicándolo.

Dependencias:
HU-02, HU-04

------------------------------------------------------------

HU-04 – Archivar convocatorias vencidas
Rol: Sistema de Gestión Académica
Quiero archivar automáticamente las convocatorias vencidas
Para mantener la lista de convocatorias actualizada

Criterios de Aceptación:
- Dado que una convocatoria supere su fecha de cierre, cuando el sistema realice la verificación, entonces debe moverla al archivo histórico.
- Dado que una convocatoria esté archivada, cuando se consulte el listado de activas, entonces no debe aparecer.
- Dado que un coordinador consulte el archivo histórico, cuando se muestre, debe incluir fechas y datos completos.

Dependencias:
HU-02, HU-03

------------------------------------------------------------

HU-05 – Editar convocatoria publicada
Rol: Coordinador Académico
Quiero editar la información de una convocatoria publicada
Para corregir o actualizar sus requisitos antes del cierre

Criterios de Aceptación:
- Dado que una convocatoria esté activa, cuando se solicite edición, entonces el sistema debe permitir modificar los campos autorizados.
- Dado que se realice una edición, cuando se guarden los cambios, entonces deben reflejarse inmediatamente.
- Dado que falte información obligatoria, cuando se intente guardar, entonces el sistema debe impedirlo.

Dependencias:
HU-01, HU-02

------------------------------------------------------------

HU-06 – Registrar postulación
Rol: Estudiante Aspirante
Quiero registrar mi postulación a una convocatoria específica
Para participar en el proceso de selección

Criterios de Aceptación:
- Dado que el estudiante esté autenticado, cuando seleccione una convocatoria activa y confirme, entonces el sistema debe registrar la candidatura.
- Dado que un estudiante intente postularse a una convocatoria cerrada, entonces debe impedirse y mostrar un mensaje de error.
- Dado que un estudiante ya esté registrado, cuando intente postularse de nuevo, entonces el sistema debe evitar duplicados.

Dependencias:
HU-01, HU-03

------------------------------------------------------------

HU-07 – Adjuntar hoja de vida
Rol: Estudiante Aspirante
Quiero adjuntar mi hoja de vida en formato digital
Para que pueda ser evaluada por el comité y el módulo de IA

Criterios de Aceptación:
- Solo se aceptan archivos PDF o DOCX.
- Rechaza archivos mayores a 5 MB.
- La hoja de vida debe mostrarse adjunta en la postulación.

Dependencias:
HU-06, HU-11

------------------------------------------------------------

HU-08 – Validar requisitos mínimos
Rol: Sistema de Gestión Académica
Quiero validar que el estudiante cumple requisitos mínimos antes de permitir su postulación
Para evitar candidaturas no aptas

Criterios de Aceptación:
- Bloquear la postulación si no cumple promedio, semestre o curso previo.
- Registrar correctamente la postulación si cumple.
- Mostrar mensaje detallado cuando no cumple requisitos.

Dependencias:
HU-06, HU-01

------------------------------------------------------------

HU-09 – Enviar confirmación de registro
Rol: Sistema de Gestión Académica
Quiero enviar una confirmación al estudiante
Para que tenga constancia de su postulación

Criterios de Aceptación:
- Enviar confirmación tras completar la postulación.
- No enviar confirmación si la postulación está incompleta.
- Mostrar la confirmación en el estado del estudiante.

Dependencias:
HU-06, HU-08

------------------------------------------------------------

HU-10 – Consultar estado de postulación
Rol: Estudiante Aspirante
Quiero consultar el estado de mi postulación
Para conocer si estoy en evaluación, preselección o descartado

Criterios de Aceptación:
- Mostrar el estado actual de la postulación.
- Actualizar el estado cuando cambie.
- Mostrar mensaje si el estudiante no tiene postulaciones.

Dependencias:
HU-06, HU-09

------------------------------------------------------------

HU-11 – Filtrar postulaciones (IA)
Rol: Módulo de IA
Quiero filtrar las postulaciones según los requisitos
Para entregar solo candidatos aptos al comité

Criterios de Aceptación:
- Mostrar solo candidatos que cumplan requisitos mínimos.
- Excluir candidatos no aptos.
- Actualizar lista cuando cambien los criterios.

Dependencias:
HU-07, HU-08

------------------------------------------------------------

HU-12 – Clasificar candidatos por puntaje
Rol: Módulo de IA
Quiero clasificar a los candidatos según su puntaje de evaluación automática
Para priorizar a los mejores perfiles

Criterios de Aceptación:
- Generar ranking de mayor a menor.
- Actualizar ranking cuando cambien criterios.
- Reflejar empates adecuadamente.

Dependencias:
HU-11, HU-07

------------------------------------------------------------

HU-13 – Consultar ranking generado por IA
Rol: Comité de Selección
Quiero consultar el ranking generado por la IA
Para iniciar la evaluación manual de los postulantes

Criterios de Aceptación:
- Mostrar la lista completa y ordenada.
- Actualizar la lista cuando varíen puntajes.
- Permitir aplicar filtros adicionales.

Dependencias:
HU-12, HU-11

HU-14 – Ajustar parámetros de IA
Rol: Coordinador Académico
Quiero ajustar los parámetros de filtrado de la IA
Para adaptarlos a diferentes convocatorias

Criterios de Aceptación:
- Actualizar automáticamente los criterios al modificarlos.
- Aplicar los nuevos parámetros en filtros posteriores.
- Mostrar la configuración actual al resto de coordinadores.
